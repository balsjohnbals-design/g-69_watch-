How to read this: Each pair of hex digits is one byte. Your original working version decoded Temp from bytes 0–1 (as a 16-bit little-endian number divided by 10), and HRV from byte 2 (single byte). Use the Tune Offsets tab to adjust if values look wrong.
Byte offset tuning: Connect your watch, then check RAW PACKETS to see what bytes are coming in. Adjust the offset numbers below to map the right bytes to each metric. Your original working values are pre-filled: Temp at 0 (UInt16 LE ÷10), HRV at 2 (UInt8).
Value ranges used for auto-detection:
Temp: 30–45 °C  |  HRV: 5–250 ms  |  HR: 40–220 bpm  |  SpO2: 70–100%
Values outside these ranges are silently ignored. If a metric never appears, widen its offset search.
